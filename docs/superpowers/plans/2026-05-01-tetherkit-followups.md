# TetherKit Follow-Ups Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land four tracked follow-ups from the prior TetherKit code review: scoped logging, USB ZLP correctness on TX, interrupt-IN endpoint handling for `RESPONSE_AVAILABLE` / `INDICATE_STATUS`, and a host-side XCTest target for the RNDIS framing/protocol math.

**Architecture:** The DriverKit dext (`TetherKitDriver.dext`) extends `IOUserNetworkEthernet`. RNDIS pure logic currently lives mixed into `TetherKitDriver.cpp`. We will (a) replace `OS_LOG_DEFAULT` with a project-tagged log handle pattern that works inside DriverKit's reduced os/log surface, (b) detect the OUT pipe's `wMaxPacketSize` at open time and pad TX transfers with one trailing zero byte when their length is an exact multiple of MPS, (c) add a third pipe — the comm-iface interrupt-IN — armed with a new `InterruptReadComplete` async callback that signals a `responseAvailable` flag the rewritten `rndisCommand` waits on, and (d) extract pure C++ protocol math (struct sizes, framing, init clamp, MAC fixup) into a new `RNDISProtocolCore.h`/`.cpp` pair that the existing dext continues to use, plus a new `TetherKitProtocolTests` macOS XCTest bundle target that links those two files alone — no DriverKit headers.

**Tech Stack:** DriverKit 22.5+, USBDriverKit, NetworkingDriverKit, C++20, Xcode 15+ project format (objectVersion 56), XCTest, `xcodebuild`.

**IMPORTANT — DriverKit constraint that re-shapes Task 1:**
`os_log_create()` is NOT exposed by DriverKit's `<os/log.h>` (verified at `/Applications/Xcode.app/.../DriverKit.sdk/System/DriverKit/usr/include/os/log.h` — only `OS_LOG_DEFAULT` and `_os_log_impl` are declared, no symbol is provided in `libSystem.tbd` either). Subsystem-tagged `os_log_t` handles cannot be created from a dext. The realistic equivalent of "make `log stream --predicate 'subsystem ==' work" is to use `--predicate 'process == "com.prostec.tetherkit.driver"'` and additionally tag every log line with a static `[TetherKit]` prefix (already partly done) plus a category prefix — both filterable by `eventMessage CONTAINS`. Task 1 below ships that approach and updates `BUILD_NOTES.md` with the correct `log stream` invocation. We do NOT introduce code that won't compile against the DriverKit SDK.

## Task 2: ZLP padding for TX transfers that are an exact multiple of wMaxPacketSize

**Files:**
- Modify: `TetherKitDriver/TetherKitDriver.cpp:441-544` (`openDataInterface`)
- Modify: `TetherKitDriver/TetherKitDriver.cpp:60-88` (`TetherKitDriver_IVars`)
- Modify: `TetherKitDriver/TetherKitDriver.cpp:729-796` (`consumeTxPackets`)
- Modify: `TetherKitDriver/RNDISProtocol.h` (raise `OUT_BUF_SIZE` to leave room for one trailing pad byte)

**Background fact for the engineer:** DriverKit's `IOUSBHostPipe` does NOT expose `kUSBHostPipeOptionZeroLengthPacket`. The IIG header at `.../USBDriverKit.framework/Headers/IOUSBHostPipe.iig` only offers `Open`, `AsyncIO`, `IO`, `Abort`, `ClearStall`, `SetIdlePolicy`, `AdjustPipe`, `GetDescriptors`. There is no pipe option flag. The standard DriverKit workaround is the same as the Linux `URB_ZERO_PACKET` substitute used by other userspace stacks: when the transfer length is a multiple of `wMaxPacketSize`, append one extra zero byte. The device parses RNDIS messages by `msg_len` from the header, so a stray trailing byte is harmless. We compute the OUT endpoint's MPS from the descriptor at `openDataInterface` time using `IOUSBGetEndpointMaxPacketSize`. (For HS bulk that's 512; for FS it's 64; for SS it's 1024.)

```cpp
        uint32_t submitLen = totalLen;
```

Add this function definition immediately after `openDataInterface` (so just before `allocateTransferResources`, around line 545):

```cpp
kern_return_t
TetherKitDriver::openInterruptPipe()
{
    // The interrupt-IN endpoint lives on the comm interface, which is already
    // Open()ed by Start_Impl(). We just need to walk its endpoint descriptors,
    // find the first interrupt-IN, copy a pipe handle, and allocate a buffer.

    if (!IVARS->commInterface) {
        return kIOReturnNotReady;
    }

    const IOUSBConfigurationDescriptor * cfgDesc =
        IVARS->commInterface->CopyConfigurationDescriptor();
    if (!cfgDesc) {
        return kIOReturnError;
    }
    const IOUSBInterfaceDescriptor * commIfDesc =
        IVARS->commInterface->GetInterfaceDescriptor(cfgDesc);
    if (!commIfDesc) {
        IOUSBHostFreeDescriptor(cfgDesc);
        return kIOReturnError;
    }

    // Walk for the first interrupt-IN endpoint on the comm interface.
    uint8_t intEpAddr = 0;
    uint16_t intEpMps = 0;
    {
        // Need device speed for MPS parsing.
        uint8_t deviceSpeed = 0;
        IOUSBHostDevice * speedDev = nullptr;
        if (IVARS->commInterface->CopyDevice(&speedDev) == kIOReturnSuccess && speedDev) {
            speedDev->GetSpeed(&deviceSpeed);
            OSSafeReleaseNULL(speedDev);
        }

        const IOUSBEndpointDescriptor * ep = nullptr;
        while ((ep = IOUSBGetNextEndpointDescriptor(
                    cfgDesc,
                    commIfDesc,
                    reinterpret_cast<const IOUSBDescriptorHeader *>(ep))) != nullptr) {
            const bool isIn = (ep->bEndpointAddress & kIOUSBEndpointDescriptorDirection) != 0;
            const bool isInterrupt =
                (ep->bmAttributes & kIOUSBEndpointDescriptorTransferType) ==
                kIOUSBEndpointDescriptorTransferTypeInterrupt;
            if (isIn && isInterrupt) {
                intEpAddr = ep->bEndpointAddress;
                intEpMps  = IOUSBGetEndpointMaxPacketSize(deviceSpeed, ep);
                break;
            }
        }
    }
    IOUSBHostFreeDescriptor(cfgDesc);

    if (intEpAddr == 0) {
        // No interrupt-IN endpoint. Some Linux gadget RNDIS profiles omit it.
        // That's OK — we just keep the legacy poll-only rndisCommand path.
        LOG_INFO("openInterruptPipe: no interrupt-IN endpoint; will poll EP0");
        return kIOReturnSuccess;
    }
    if (intEpMps == 0) {
        intEpMps = 16; // Spec minimum for FS interrupt EPs is small; safe default.
    }

    kern_return_t ret = IVARS->commInterface->CopyPipe(intEpAddr, &IVARS->intPipe);
    if (ret != kIOReturnSuccess || !IVARS->intPipe) {
        return ret != kIOReturnSuccess ? ret : kIOReturnError;
    }

    IVARS->intBufLen = intEpMps;
    ret = IOBufferMemoryDescriptor::Create(kIOMemoryDirectionIn,
                                           IVARS->intBufLen, 0, &IVARS->intBuf);
    if (ret != kIOReturnSuccess || !IVARS->intBuf) {
        return ret != kIOReturnSuccess ? ret : kIOReturnNoMemory;
    }
    uint64_t mappedAddress = 0;
    uint64_t mappedLength = 0;
    ret = IVARS->intBuf->Map(0, 0, 0, 0, &mappedAddress, &mappedLength);
    if (ret != kIOReturnSuccess) {
        return ret;
    }
    IVARS->intBufAddr = mappedAddress;

    ret = CreateActionInterruptReadComplete(0, &IVARS->intAction);
    if (ret != kIOReturnSuccess) {
        return ret;
    }
    return kIOReturnSuccess;
}
```

```cpp
kern_return_t
TetherKitDriver::armInterruptRead()
{
    if (!IVARS->intPipe || !IVARS->intBuf || !IVARS->intAction) {
        return kIOReturnSuccess; // No pipe means we silently fall back to polling.
    }
    // completionTimeoutMs MUST be 0 for interrupt endpoints (per IOUSBHostPipe.iig).
    kern_return_t ret = IVARS->intPipe->AsyncIO(IVARS->intBuf,
                                                 IVARS->intBufLen,
                                                 IVARS->intAction,
                                                 0);
    if (ret == kIOReturnSuccess) {
        IVARS->intArmed = true;
    } else if (!isUsbGoneStatus(ret)) {
        LOG_ERROR("armInterruptRead: AsyncIO failed: %08x", ret);
    }
    return ret;
}
```

- [ ] **Step 8: Implement `handleNotification`**

Add the CDC notification parser immediately after `armInterruptRead`:

```cpp
// CDC interrupt-IN notifications use the layout:
//   bmRequestType (1) | bNotification (1) | wValue (2) | wIndex (2) | wLength (2) [| data]
// RNDIS-over-USB shortcuts this to: a single 8-byte payload of two LE uint32s.
//   word0 = notification code, word1 = reserved
// HoRNDIS treats only RESPONSE_AVAILABLE specially; INDICATE_STATUS arrives in-band
// as a control-channel message, not as an interrupt notification — but some Android
// quirks deliver MEDIA_CONNECT/DISCONNECT here too. We handle both robustly.

static constexpr uint32_t RNDIS_NOTIF_RESPONSE_AVAILABLE = 0x00000001;

void
TetherKitDriver::handleNotification(const uint8_t * buf, uint32_t len)
{
    if (len < 8 || !buf) {
        return;
    }
    // Read first 32-bit LE word.
    uint32_t code = (uint32_t) buf[0]
                  | ((uint32_t) buf[1] << 8)
                  | ((uint32_t) buf[2] << 16)
                  | ((uint32_t) buf[3] << 24);
    if (code == RNDIS_NOTIF_RESPONSE_AVAILABLE) {
        IVARS->responseAvailable = true;
        LOG_DEBUG("interrupt: RESPONSE_AVAILABLE");
        return;
    }

    // INDICATE_STATUS arrives over the control channel as an RNDIS message, not
    // here — but if a device incorrectly muxes link state into the interrupt
    // notification stream, we treat the codes Microsoft documents as link events:
    //   0x4001000B RNDIS_STATUS_MEDIA_CONNECT   -> link up
    //   0x4001000C RNDIS_STATUS_MEDIA_DISCONNECT-> link down
    if (code == 0x4001000Bu) {
        LOG_INFO("interrupt: MEDIA_CONNECT (link up)");
        reportLinkStatus(kIOUserNetworkLinkStatusActive, 0);
    } else if (code == 0x4001000Cu) {
        LOG_INFO("interrupt: MEDIA_DISCONNECT (link down)");
        reportLinkStatus(kIOUserNetworkLinkStatusInactive, 0);
    } else {
        LOG_DEBUG("interrupt: unknown notification code 0x%08x", code);
    }
}
```

```cpp
    // endpoint, then issue GET_ENCAPSULATED_RESPONSE. Some devices (R6 in our
    // research notes) won't reply to GET_ENCAPSULATED_RESPONSE until they've

                sawNotification = true;
                break;

                return kIOReturnAborted;

            LOG_ERROR("rndisCommand: timed out waiting for RESPONSE_AVAILABLE");
            // Fall through to the polling attempts below — some devices skip
            // the notification entirely.
        }

        sawNotification ? kPostNotifyAttempts : static_cast<uint32_t>(kMaxAttempts);

        uint16_t bytesReceived = 0;

            static_cast<uint16_t>(RNDIS_CMD_BUF_SZ),

            &bytesReceived,

            return ret;
        }
        if (static_cast<uint32_t>(bytesReceived) > RNDIS_CMD_BUF_SZ) {
            LOG_ERROR("rndisCommand: response %u exceeds buffer %u", bytesReceived, RNDIS_CMD_BUF_SZ);
            return kIOReturnIOError;

        if (static_cast<uint32_t>(bytesReceived) < kMinResponseBytes) {

            continue;

            continue;

        if (hdr->request_id != requestId) {
```

```cpp
/* RNDISProtocolCore.cpp
 * Pure C++ implementation of the testable RNDIS framing math.
 */

#include "RNDISProtocolCore.h"

namespace rndis {

ParseResult parseInboundBuffer(const uint8_t * buf,
                               uint32_t        size,
                               PacketCallback  cb,
                               void *          context)
{
    while (size > 0) {
        if (size < sizeof(rndis_data_hdr)) {
            return ParseResult::TRUNCATED_HEADER;
        }
        const auto * hdr = reinterpret_cast<const rndis_data_hdr *>(buf);
        const uint32_t msgLen  = le32_to_cpu(hdr->msg_len);
        const uint32_t dataOfs = le32_to_cpu(hdr->data_offset);
        const uint32_t dataLen = le32_to_cpu(hdr->data_len);

        if (hdr->msg_type != RNDIS_MSG_PACKET) {
            return ParseResult::WRONG_MSG_TYPE;
        }
        if (msgLen > size || msgLen < sizeof(rndis_data_hdr)) {
            return ParseResult::INVALID_MSG_LEN;
        }
        // Data is at +8+data_offset and spans data_len bytes; the entire
        // window must fit inside this RNDIS message.
        if (dataOfs > msgLen ||
            dataLen > msgLen ||
            (8u + dataOfs + dataLen) > msgLen) {
            return ParseResult::INVALID_DATA_BOUNDS;
        }
        if (dataLen > 0 && cb) {
            cb(context, buf + 8u + dataOfs, dataLen);
        }
        size -= msgLen;
        buf  += msgLen;
    }
    return ParseResult::OK;
}

uint32_t clampMaxTransferSize(uint32_t deviceReportedMax, uint32_t outBufPayloadMax)
{
    uint32_t v = (deviceReportedMax < outBufPayloadMax) ? deviceReportedMax : outBufPayloadMax;
    const uint32_t floor =
        static_cast<uint32_t>(sizeof(rndis_data_hdr)) + ETHERNET_MTU + 14u;
    if (v < floor) {
        v = (floor < outBufPayloadMax) ? floor : outBufPayloadMax;
    }
    return v;
}

bool fixupMacMulticastBit(uint8_t mac[6])
{
    if ((mac[0] & 0x01u) == 0) {
        return false;
    }
    mac[0] = static_cast<uint8_t>((mac[0] & 0xFEu) | 0x02u);
    return true;
}

} // namespace rndis
```
